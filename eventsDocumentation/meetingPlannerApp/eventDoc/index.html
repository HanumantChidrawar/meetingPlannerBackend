Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
online-user-list event
This event ("online-user-list") has to be listened on the user's end to identify all available Normal(non-admin) users that are currently online.
The output will be an object, object has key as userId and value as full name.
auth-error event
This event ("auth-error") has to be listened to know if any error has occurred on socket.
This event is emitted to every user of Meeting Planner Application.
userId event
This event ("userId") has to be listened to get the notification when any meeting is Scheduled/Updated/Deleted.
Every user has to listen on its own userId for the meeting notifications.

Events to emit

set-user event
This event ("set-user") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token.
The following data has to be emitted:
authentication token
eg- ("set-user",authToken);
notify-user event
This event ("notify-user") has to be emitted when a meeting has Scheduled/Update/Deleted.
This event will notify with user with whom the meeting is Scheduled/Updated/Deleted. The following data has to be emitted:
eg- data = { userId:"Id od the user who is to be notified", toast:"Message which is to be shown to user" };
("notify-user",data);
email-notification event
This event ("email-notification") has to be emitted to remind user of the upcoming meeting through email.
This event will notify with user with email to get ready for upcoming meeting. The following data has to be emitted:
-- If you want to notify the admin as well as participating user then pass the meetingDetails object by setting property "email" Email of admin and notifyUser= "true".
-- If you want to notify the admin for the meeting then pass the meetingDetails object by seeting the property "email" as email of the admin.
-- If you want to notify the Particapant of the meeting that meeting has Scheduled then pass the MeetingDetails object with property "type" set as "create".
-- If you want to notify the Particapant of the meeting that meeting has Re-Scheduled then pass the MeetingDetails object with property "type" set as "update".
-- If you want to notify the Particapant of the meeting that meeting has Cancelled then pass the MeetingDetails object with property "type" set as "delete".
-- If you want to notify the particpant that the meeting will start soon then don't set any of the above property on meeting details object.
eg- ("email-notification",meetingDetails);
disconnection
This event ("disconnection") has to be emitted, when the user gets disconnected(logout).
eg- ("disconnect","");
